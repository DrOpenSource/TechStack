# AI VibeCoding - Mobile-First Development Platform

A Next.js 14 application for building applications through AI-powered conversations, optimized for mobile devices.

## Features

- **Mobile-First Design**: Optimized for touch interactions and mobile screens
- **AI-Powered Development**: Chat with AI to build your applications
- **Real-Time Preview**: See your app come to life as you build it
- **PWA Support**: Install as a native app on your mobile device
- **Mock Mode**: Demo the app without API keys using faker.js
- **Code Export**: Export your projects as ZIP files
- **Multi-Provider**: Support for Claude and Gemini AI models

## Tech Stack

- **Framework**: Next.js 14 with App Router
- **Language**: TypeScript
- **Styling**: TailwindCSS with mobile-first breakpoints
- **UI Components**: shadcn/ui
- **State Management**: Zustand with persistence
- **Animations**: Framer Motion
- **Icons**: Lucide React
- **Mock Data**: Faker.js
- **PWA**: next-pwa
- **Code Export**: JSZip

## Getting Started

### Prerequisites

- Node.js 18+ and npm/yarn/pnpm
- Git

### Installation

1. Clone the repository:
```bash
git clone <repository-url>
cd TechStack
```

2. Install dependencies:
```bash
npm install
# or
yarn install
# or
pnpm install
```

3. Set up environment variables:
```bash
cp .env.example .env.local
```

4. Run the development server:
```bash
npm run dev
# or
yarn dev
# or
pnpm dev
```

5. Open [http://localhost:3000](http://localhost:3000) in your browser

## Mock Mode

The application runs in mock mode by default (`NEXT_PUBLIC_MOCK_MODE=true`). This allows you to:

- Test the application without API keys
- See realistic AI responses generated by faker.js
- Explore all features without costs
- Demo the application safely

### Mock Features

- **AI Responses**: Context-aware mock responses based on user input
- **Projects**: Automatically generated project templates
- **User Authentication**: Simplified login with any email
- **Code Generation**: Sample TypeScript/React code snippets

## Environment Variables

Create a `.env.local` file based on `.env.example`:

```env
# Mock Mode - Set to false to use real AI providers
NEXT_PUBLIC_MOCK_MODE=true

# AI Provider API Keys (only needed if mock mode is disabled)
NEXT_PUBLIC_CLAUDE_API_KEY=your_claude_key
NEXT_PUBLIC_GEMINI_API_KEY=your_gemini_key

# App Configuration
NEXT_PUBLIC_APP_NAME="AI VibeCoding"
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Feature Flags
NEXT_PUBLIC_ENABLE_PWA=true
NEXT_PUBLIC_ENABLE_CODE_EXPORT=true
NEXT_PUBLIC_ENABLE_PREVIEW=true
```

## State Management

The app uses Zustand for state management with the following stores:

### User Store (`user-store.ts`)
- User authentication state
- Profile information
- Session management

### Project Store (`project-store.ts`)
- Project list
- Active project
- Project files
- CRUD operations

### Chat Store (`chat-store.ts`)
- Chat messages
- Message history
- Conversation context

### AI Provider Store (`ai-provider-store.ts`)
- Selected AI provider (Claude/Gemini)
- Model configuration
- Mock mode toggle
- API settings


Example:
```bash
npx shadcn-ui@latest add card
npx shadcn-ui@latest add dialog
npx shadcn-ui@latest add dropdown-menu
```

## Development Tips

1. **Mobile Testing**: Use Chrome DevTools mobile emulation or test on real devices
2. **Mock Data**: Customize mock responses in `lib/mock/ai-responses.ts`
3. **Styling**: Use Tailwind's mobile-first approach (`sm:`, `md:`, `lg:`)
4. **State**: All stores use persistence, clear localStorage to reset
5. **Icons**: Generate PWA icons using https://realfavicongenerator.net/
